# Conference Attendance Optimizer - Gemini Guidelines

## Build & Test Commands
- **Build**: `npm run build` (TBD)
- **Test**: `npm test` (TBD)
- **Lint**: `npm run lint` (TBD)

## Project Structure
- `.ai/`: AI context, history, and PRP artifacts.
- `AGENT.md`: High-level AI engagement rules.
- `README.md`: Project documentation.
- `src/`: Source code (to be created).

## PRP Workflow
1. **PRD**: Define requirements in `.ai/PRPs/prds/`.
2. **PLAN**: Create implementation plan in `.ai/PRPs/plans/`.
3. **REPORT**: Document completion in `.ai/PRPs/reports/`.
4. **SESSION**: Update `.ai/history/` after every major task.

## Standards
- Follow **Git Flow**.
- Use descriptive commit messages.
- Update `state.md` and `decisions.md` in `.ai/context/` when changed.
